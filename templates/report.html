Executive Summary

Content Security Policy (CSP) is a browser-enforced security mechanism that restricts the sources from which a web application can load and execute content such as scripts, styles, images, and other resources. When configured correctly, CSP significantly reduces the attack surface for client-side vulnerabilities, particularly cross-site scripting (XSS) and data injection attacks.

SmartCSP aims to simplify the adoption of strong CSP configurations by automatically analyzing real resource usage on a target website and generating a tailored policy that balances security with functionality. Instead of relying on overly permissive defaults, SmartCSP derives precise allowlists and validates the resulting policy through controlled sandbox testing.

Deployment Instructions

1. Deployment

To deploy the generated Content Security Policy (CSP), first copy the CSP header produced by SmartCSP: CSP Generator, Tester, and Optimizer exactly as shown in this report. This header defines the security rules that the browser must enforce when loading and executing resources.

The CSP must be included as an HTTP response header and can be configured either at the application backend or at the web server level, depending on how the website is deployed.

Method 1: Backend Application

For dynamically generated websites, the CSP should be attached directly to the HTTP response by the backend framework before the page is sent to the browser.

Flask (Python)
from flask import make_response, render_template @app.route("/") def index(): response = make_response(render_template("index.html")) response.headers["Content-Security-Policy"] = ( "PASTE GENERATED CSP HERE" ) return response
Node.js (Express)
app.use((req, res, next) => { res.setHeader( "Content-Security-Policy", "PASTE GENERATED CSP HERE" ); next(); });

Method 2: Web Server Configuration

For static websites or reverse-proxy setups, the CSP can be configured directly in the web server configuration so that it is applied globally to all responses.

Nginx
add_header Content-Security-Policy "<PASTE GENERATED CSP HERE>";

After updating the configuration, the Nginx service must be restarted for the changes to take effect.

Apache
Header set Content-Security-Policy "< PASTE GENERATED CSP HERE >"

After updating the configuration, the Apache service must be restarted to apply the CSP header.

2. Testing

Before enforcing the CSP in production, it is recommended to deploy the policy in report-only mode. This allows violations to be logged without blocking resources.

Content-Security-Policy-Report-Only: ...

3. Verification

After deployment, the CSP should be verified using browser developer tools.

  1. Open the website in a modern browser.
  2. Open developer tools and navigate to the Network tab.
  3. Select the main document request.
  4. Inspect the Response Headers section.

If the policy is applied correctly, the Content-Security-Policy header will be visible and any violations will appear in the browser console.

Nonce Usage Guide

A nonce (number used once) is a cryptographically secure random value that allows specific inline scripts to execute under a strict Content Security Policy. By requiring a nonce, the browser ensures that only explicitly authorized scripts are permitted to run.

SmartCSP does not generate or embed nonces automatically, as nonce values must be created securely on the server for each HTTP response. Including a static or reused nonce would weaken the security guarantees provided by CSP and could be exploited by attackers.

Generating a Secure Nonce

A new nonce must be generated for every request using a cryptographically secure random number generator.

import secrets import base64 def generate_nonce(): random_bytes = secrets.token_bytes(16) return base64.b64encode(random_bytes).decode("utf-8")

Including the Nonce in the CSP Header

The generated nonce must be included in the Content Security Policy header so that the browser knows which inline scripts are allowed to execute.

Content-Security-Policy: script-src 'self' 'nonce-{RANDOM_NONCE}'; object-src 'none';

5. Applying the Nonce to Inline Scripts

The same nonce value used in the CSP header must also be applied to the corresponding inline script elements in the HTML.

<script nonce="{RANDOM_NONCE}"> // Inline JavaScript code </script>
