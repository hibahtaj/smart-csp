What is Content Security Policy?

Think of a website like a house. Every script, image, or stylesheet that loads on the page is like a visitor entering that house. Content Security Policy (CSP) acts like a security guard at the door. It tells the browser exactly which sources are trusted and blocks everything else. If an attacker tries to inject malicious code, the browser simply refuses to run it. Without CSP, websites become vulnerable to attacks like Cross-Site Scripting (XSS) where attackers inject scripts that run in a user's browser.

What Happens Without CSP?

XSS Attacks

Attackers inject malicious scripts that steal user data or cookies.

Data Theft

Hackers can secretly send login credentials to external servers.

Website Defacement

Attackers can modify content or redirect users to harmful sites.

Malware Injection

Users may unknowingly download malware from compromised scripts.
